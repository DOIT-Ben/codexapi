# Manual Recharge Runbook

这套部署默认走人工充值：

1. 用户注册账号。
2. 用户把注册邮箱发给你。
3. 你确认转账。
4. 你进后台用户管理，搜索邮箱。
5. 给用户加余额，或进入该用户的 API Key 给 key 加额度。
6. 用户登录后自行查看余额、额度、密钥和用量。

建议规则：

- 新注册用户默认余额为 0。
- 不要帮用户共用一把 key。
- 每次充值都备注金额、时间、转账渠道。
- 额度异常消耗时，先暂停该用户 key，再查 usage 日志。
- 用户需要更高并发时，单独调整他的并发或 RPM。

给用户的调用信息：

```text
Base URL: https://api.example.com/v1
API Key: 用户自己的 sk-...
Model: 后台分组里开放的模型
```
